Technical Deep-dive
Dynamic Workflows Anthropic / claude.com
5 patterns Claude dùng khi tự viết harness cho task phức tạp
  • fan-out-synthesize: chia task thành nhiều subagents song song, barrier step tổng hợp kết quả — tránh cross-contamination.
  • adversarial-verify: mỗi subagent sinh output có một "verifier agent" đối nghịch review — combat self-preferential bias.
  • tournament: N agents thử đồng thời, judge agent so sánh pairwise cho đến khi có winner — tốt cho naming / design.
  • loop-until-done: spawn agents đến khi stop condition (no new findings, all errors resolved) — không fixed iteration count.
  • classify-and-route: classifier agent đọc task, chọn model phù hợp (Sonnet vs Opus) dựa trên complexity thực tế.

Agent Memory O'Reilly Radar
3 tầng memory trong production agent — và lúc nào cần từng tầng
  • in-context (tier 1): conversation history trong prompt — đủ cho short sessions; context rot khi >70% window đầy.
  • vector-search (tier 2): pgvector + Postgres là default 2026; thêm khi history vượt context limit — không cần dedicated DB.
  • persistent-memory (tier 3): Mem0 / Letta / Zep cho cross-session learning; thiết kế memory governance từ đầu vì memory drift là failure mode thực tế.
  • "context engineering" thay thế "prompt engineering" làm core discipline — kiến trúc những gì agent thấy mỗi lần gọi.

Long-running Agents O'Reilly Radar / Addy Osmani
Kiến trúc brain/hands/session: 3 tường mà mọi long-running agent đều gặp
  • finite-context: context rot bắt đầu sớm hơn hard limit; full context reset (không phải summarization) là fix đúng cho runs > vài giờ.
  • session-as-event-log: append-only log là primitive quan trọng nhất — không có nó, container failure = session failure. Anthropic báo cáo p95 latency giảm 90% khi decoupling brain/hands.
  • done-condition-first: viết điều kiện kết thúc trước khi agent bắt đầu — leverage cao nhất để tránh agent "done" sớm.
  • Cursor's planner/worker/judge split: planners spawn subplanners, workers chỉ execute, judges quyết khi nào xong — mô hình đang chạy production.

MCP Security O'Reilly · Endor Labs
MCP đã thắng protocol war — nhưng security là vấn đề mở còn lại
  • Endor Labs phân tích 2,614 MCP servers: 82% prone to path traversal, 67% prone to code injection.
